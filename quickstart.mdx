This guide will walk you through running the parser for the first time and understanding the output.

## Prerequisites

Before starting, make sure you have:

- [.NET 8 SDK installed](/installation#prerequisites)
- [Cloned the repository](/installation#installation-steps)
- An active internet connection

## Running the Parser

<Steps>
  <Step title="Navigate to the Project Directory">
    Open a terminal and navigate to the cloned repository:
    
    ```bash
    cd ProletarioParasitoPodcastParser
    ```
  </Step>
  
  <Step title="Run the Application">
    Execute the parser using the .NET CLI:
    
    ```bash
    dotnet run
    ```
    
    <Info>
      The first run may take a few seconds as .NET restores dependencies and compiles the project.
    </Info>
  </Step>
  
  <Step title="View the Output">
    The parser will fetch the RSS feed, parse all episodes, and display a summary:
    
    ```
    [INFO] Fetching RSS feed from https://anchor.fm/s/fae337f8/podcast/rss
    [INFO] Fetched 142 RSS feeds
    
    === PARSE RESULTS SUMMARY ===
    
        Total Episodes: 142
        With Guests: 38
        Special Episodes: 21
    ```
    
    <Note>
      Episode counts will vary depending on the current state of the podcast feed. The output shown here is an example.
    </Note>
  </Step>
</Steps>

## Understanding the Output

The parser displays information in a structured, color-coded format:

### Information Messages

```
[INFO] Fetching RSS feed from https://anchor.fm/s/fae337f8/podcast/rss
[INFO] Fetched 142 RSS feeds
```

These messages show:
- The RSS feed URL being accessed
- The number of episode items retrieved from the feed

### Parse Results Summary

```
=== PARSE RESULTS SUMMARY ===

    Total Episodes: 142
    With Guests: 38
    Special Episodes: 21
```

The summary provides:

| Metric | Description |
|--------|-------------|
| **Total Episodes** | Total number of episodes found in the RSS feed |
| **With Guests** | Number of episodes featuring one or more guest speakers |
| **Special Episodes** | Episodes marked as specials, extras, or bonus content |

## How the Parser Works

Under the hood, the parser follows a simple pipeline:

<Steps>
  <Step title="Fetch RSS Feed">
    The `RssFeedService` downloads the podcast's RSS feed from Anchor.fm using HTTP.
    
    ```csharp
    var rssEpisodes = await rssFeedService.GetEpisodesAsync();
    ```
  </Step>
  
  <Step title="Parse Episodes">
    The `EpisodeParserService` processes each episode to extract:
    
    - Episode number
    - Cleaned title (without podcast suffix)
    - Guest names (if present)
    - Special episode flag
    - Simpsons episode references from descriptions
    - Duration in seconds
    
    ```csharp
    var allParsed = rssEpisodes.Select(e => parserService.ParseEpisode(e)).ToList();
    ```
  </Step>
  
  <Step title="Display Results">
    The `ConsoleOutputService` formats and displays the summary with color-coded output:
    
    ```csharp
    outputService.ShowSumary(allParsed);
    ```
  </Step>
</Steps>

## Title Parsing Examples

The parser intelligently handles various title formats:

<CodeGroup>
```text Standard Episode
42 - El capitalismo | Proletario y Parásito

Parsed as:
- Episode: 42
- Title: El capitalismo
- Guest: None
- Special: No
```

```text Episode with Guest
42 - El capitalismo (con Juan Garcia) | Proletario y Parásito

Parsed as:
- Episode: 42
- Title: El capitalismo
- Guest: Juan Garcia
- Special: No
```

```text Multiple Guests
15 - La economía (con Ana López, Pedro Ruiz y María Santos) | Proletario y Parásito

Parsed as:
- Episode: 15
- Title: La economía
- Guests: Ana López, Pedro Ruiz, María Santos
- Special: No
```

```text Special Episode
ESPECIAL #5 | Proletario y Parásito

Parsed as:
- Episode: 5
- Title: ESPECIAL #5
- Guest: None
- Special: Yes
```
</CodeGroup>

## What Gets Extracted

For each episode, the parser extracts:

<CardGroup cols={2}>
  <Card title="Episode Metadata" icon="info">
    - Episode number
    - Title (cleaned)
    - Publication date
    - Duration (converted to seconds)
  </Card>
  
  <Card title="Guest Information" icon="users">
    - Guest names (parsed from title)
    - Support for multiple guests
    - Comma and "y"-separated lists
  </Card>
  
  <Card title="Simpsons References" icon="tv">
    - Season number
    - Episode number
    - Episode title
    - Extracted from HTML descriptions
  </Card>
  
  <Card title="Special Flags" icon="flag">
    - Special episode detection
    - Bonus content identification
    - Extra episode marking
  </Card>
</CardGroup>

## Error Handling

If something goes wrong, the parser will display an error message:

<Warning>
  **Common errors:**
  - Network connection failures (can't reach RSS feed)
  - XML parsing errors (malformed feed)
  - Unexpected feed format changes
</Warning>

The parser uses a try-catch block to gracefully handle errors:

```csharp
try
{
    var rssEpisodes = await rssFeedService.GetEpisodesAsync();
    var allParsed = rssEpisodes.Select(e => parserService.ParseEpisode(e)).ToList();
    
    outputService.ShowSumary(allParsed);
} 
catch (Exception ex)
{
    outputService.ShowError(ex);
}
```
